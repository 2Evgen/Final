### 1. **Перечень автоматизируемых сценариев**

- Переход с главной страницы на страницу формы заявки на курс "Тестировщик" через каталог курсов раздел "Программирование".
- Переход с главной страницы на страницу формы заявки на курс "Тестировщик" через "Каталог курсов".
- Переход с главной страницы на страницу каталога курсов раздела "Программирование" через раздел "Направления обучения".
- Переход с главной страницы на страницу каталога курсов раздела "Программирование" через раздел "Выберите вектор развития" .
- Переход с главной страницы на страницу формы заявки на курс "Тестировщик" через раздел "Студенты".
- Переход с футера главной страницы на страницу "Каталог курсов".
- Переход сверху страницы курса "Тестировщик ПО" к форме записи на курс.
- Проверка появление кнопки "Записаться" и перехода по ней с header страницы курса "Тестировщик ПО" к форме записи на курс.
- Заполнение и отправка формы записи на курс "Тестировщик ПО" с валидными данными.

1. заполнение поля "Имя" ("Михаил")
2. заполнение поля "Телефон" ("+7(111)111-11-11")
3. клик по кнопке "Записаться"

_Ожидаемый результат:_ форма успешно отправилась, уведомление появилось.
- Отправка пустой формы.
_Ожидаемый результат_: форма не отправляется, под полями сообщение об ошибке "Обязательное поле"
- Отправка формы с одним из пустых (не заполненных) полей.

1. заполнение поля "Телефон" валидным значением (например, +7(111)111-11-11)
2. клик по кнопке "Записаться"

_Ожидаемый результат:_ форма не отправляется, под полем "Имя" сообщение об ошибке "Обязательное поле"

1. заполнение поля "Имя" валидным значением (например, "Михаил")
2. клик по кнопке "Записаться"

_Ожидаемый результат:_ форма не отправляется, под полем "Телефон" сообщение об ошибке "Обязательное поле"
- Валидация каждого из полей, в случае ввода не валидных данных, проверка на отображение соответствующих ошибок.

1. заполнение поля "Имя" валидным значением (например, "Михаил")
2. заполнение поля "Телефон" невалидным значением (например, "12345")
3. клик по кнопке "Записаться"

_Ожидаемый результат:_ форма не отправляется, под полем "Телефон" сообщение об ошибке "Номер в формате +9(999)999-99-99"

1. заполнение поля "Имя" невалидным значением (например, "12345,")
2. заполнение поля "Телефон" (например, +7(111)111-11-11)
3. клик по кнопке "Записаться"

_Ожидаемый результат_: форма не отправляется, под полем "Имя" сообщение об ошибке "Должно состоять из букв
- Отправка формы с данными из ранее успешно отправленой формы.

1. заполнение поля "Имя" валидным значением (например, "Михаил,")
2. заполнение поля "Телефон" (например, +7(111)111-11-11)
3. клик по кнопке "Записаться"

_Ожидаемый результат_: форма не отправляется, появляется уведомление об ошибке. 

**Сценарии тестирования API**:

- Проверка статуса 200 и совпадения валидных данных отправленных через UI формы записи на курс "Тестировщик ПО" с данными в body POST-запроса к серверу и новыми данными из БД.
- Проверка статуса 500 и отсутствие новых записей в БД при отправке пустого body в POST-запросе отправки формы записи на курс "Тестировщик ПО".
- Проверка статуса 500 и отсутствие новых записей в БД при отправке пустого значения у атрибута phone в body в POST-запросе отправки формы.
- Проверка статуса 500 и отсутствие новых записей в БД при отправке пустого значения у атрибута name в body в POST-запросе отправки формы.

### 2.**Перечень используемых инструментов с обоснованием выбора.**


- IDE: IntelliJ IDEA
- Язык программирования: Java
- Система сборки: Gradle 
- Тестовая среда: TestNG, может разбивать тесты на наборы тестов (suite) и имеет больше возможностей по параметризации тестов.
- Система репортинга: Allure 
- Фреймворк для UI тестирования: Selenide - удобный инструмент с автонастройкой WebDriver.
- Контроль версий: GitHubт- обмен файлами с командой
- Фреймворк для API тестирования: REST Assured вместе с JSON Schema Validator 



### 3.**Перечень необходимых разрешений, данных и доступов.**

- Разрешение на проведение автотестирования.
- Доступ к существующим тестовым данным (если таковые имеются).
- Согласованный тест-план.


### 4.**Перечень и описание возможных рисков при автоматизации.**

- "Ложные" запросы к менеджерам на обратный звонок
- Отсутствие необходимого ПО.
- Большая нагрузка на сервер.
- Не актуальность тестов, если переход был совершен из рекламы. 
- Трудности при поиске локаторов элементов на страницах.


### 5.**Перечень необходимых специалистов для автоматизации.**

- QA специалист, обладающий знанием автотестирования.
