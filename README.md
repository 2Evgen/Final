1. **Перечень автоматизируемых сценариев**

- Переход с главной страницы на страницу формы заявки на курс "Тестировщик" через каталог курсов раздел "Программирование".
- Переход с главной страницы на страницу формы заявки на курс "Тестировщик" через "Каталог курсов".
- Переход с главной страницы на страницу каталога курсов раздела "Программирование" через раздел "Направления обучения".
- Переход с главной страницы на страницу каталога курсов раздела "Программирование" через раздел "Выберите вектор развития" .
- Переход с главной страницы на страницу формы заявки на курс "Тестировщик" через раздел "Студенты".
- Переход с футера главной страницы на страницу "Каталог курсов".
- Переход сверху страницы курса "Тестировщик ПО" к форме записи на курс.
- Проверка появление кнопки "Записаться" и перехода по ней с header страницы курса "Тестировщик ПО" к форме записи на курс.


2.**Перечень используемых инструментов с обоснованием выбора.**


- IDE: IntelliJ IDEA
- Язык программирования: Java
- Система сборки: Gradle 
- Тестовая среда: TestNG, может разбивать тесты на наборы тестов (suite) и имеет больше возможностей по параметризации тестов.
- Система репортинга: Allure 
- Фреймворк для UI тестирования: Selenide - удобный инструмент с автонастройкой WebDriver.
- Контроль версий: GitHubт- обмен файлами с командой



3.**Перечень необходимых разрешений, данных и доступов.**

- Разрешение на проведение автотестирования.
- Доступ к существующим тестовым данным (если таковые имеются).
- Согласованный тест-план.


4.**Перечень и описание возможных рисков при автоматизации.**

- "Ложные" запросы к менеджерам на обратный звонок
- Отсутствие необходимого ПО.
- Большая нагрузка на сервер.
- Не актуальность тестов, если переход был совершен из рекламы. 


5.**Перечень необходимых специалистов для автоматизации.**

- QA специалист, обладающий знанием автотестирования.


6. **Интервальная оценка с учётом рисков в часах.**

- Написание тестов с учетов рисков 30 часов
